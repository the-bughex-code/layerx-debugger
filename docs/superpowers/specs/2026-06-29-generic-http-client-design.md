## Problem

The debugger only captures network traffic that flows through one of its hooks:
`LayerXHttp` (static wrapper), `LayerXDioInterceptor` (Dio), or a manual
`LayerXNetworkLogger.record(...)` call.

Real LayerX apps generated by `layerx_generator` (verified in **FesoRide** and
**SylorKey**) ship an identical `https_calls.dart` that sends every request
through a single custom `IOClient`:

```dart
late final IOClient _client = IOClient(HttpClient()...);
// _sendRequest -> _client.get / .post / .put / .patch / .delete / .send(multipart)
```

That traffic never reaches a LayerX hook, so the in-app viewer shows UI / route /
crash issues but **no API issues**. This is the root cause the user reported.

## Approach (chosen)

Add a transparent `http.BaseClient` decorator, `LayerXHttpClient`, that wraps
**any** inner `http.Client` (including `IOClient`) and logs every exchange.
Because `http.Client.get/post/put/patch/delete` and multipart `.send()` all
funnel through `BaseClient.send()`, wrapping the one client captures 100% of API
traffic â€” including multipart â€” with no per-call instrumentation.

Host app changes exactly one line:

```dart
// before
late final IOClient _client = IOClient(HttpClient()...);
// after
late final http.Client _client = LayerXHttpClient(IOClient(HttpClient()...));
```

Rejected alternatives: documenting a manual `record()` call (high per-app
friction); enhancing the setup CLI to auto-patch the custom client (fragile on
the 600â€“800 line generated files).

## Design

New file `lib/src/services/network/layerx_http_client.dart`:

- `class LayerXHttpClient extends http.BaseClient`
- `LayerXHttpClient([http.Client? inner])` â€” defaults to `http.Client()`.
- `send(http.BaseRequest request)`:
  1. Start a `Stopwatch`; capture method, URL, request headers, and request body
     (best-effort: `request.body` when it is an `http.Request`; `null` for
     multipart/streamed).
  2. `await inner.send(request)`, then **buffer** the response stream to bytes so
     the body can be logged without consuming the caller's stream.
  3. Call `LayerXNetworkLogger.record(...)` with status, decoded body, request +
     response headers and duration.
  4. Return a **new** `http.StreamedResponse` built from the buffered bytes
     (status, headers, reasonPhrase, isRedirect, persistentConnection preserved)
     so the caller receives an intact response.
  5. On error: `LayerXNetworkLogger.recordException(...)`, then `rethrow`.
  6. Whole body guarded â€” logging never breaks or alters the request.
- `close()` delegates to `inner.close()`.

Export from `lib/layerx_debugger.dart`.

## Tests (`test/network/http_client_test.dart`, runnable via `--platform chrome`)

- GET routed through an inner `MockClient` is captured (status, level=success,
  endpoint) **and** the caller still receives the intact body.
- POST request body is captured and secrets are masked.
- A thrown transport error is recorded (error level) and rethrown to the caller.
